## The Real Problem

Most “second brains” are just libraries.

Notes accumulate.  
Threads break.  
Projects lose momentum.  
Continuity decays.

**PercePhal exists to solve the continuity problem** — not storage, not search, but the ability to resume real intellectual work after any interruption.

## What PercePhal Is

A complete, ready-to-use **Method Brain** package designed for high-responsibility professionals (accounting, audit, tax, legal, compliance, AI governance, regulated advisory).

It is:
- A governed cognitive prosthesis
- Project-state-first architecture
- Continuity layer + synthesis scaffolding
- Institutional-grade traceability and governance

**It is not:**
- A generic productivity system
- A note-collection tool
- An AI replacement for thinking

## Core Standard of Success

The system is working when it can answer — quickly and credibly:

- What projects are alive?
- What is each project trying to resolve?
- What tensions are active?
- What materials matter right now?
- What changed since the last meaningful pass?
- What is the next synthesis move?

## Quick Start (Obsidian)

1. **Download the package**
   ```bash
   git clone https://github.com/pharos-ai/percephal.git
   # or download PHAROS - Second Brain.zip
   ```

2. **Install in Obsidian**
   - Open Obsidian → **”Open folder as vault”**
   - Select the `PHAROS - Second Brain` folder *(or the root of this repo)*

3. **Enable the Core Surfaces**
   - Copy the templates from `11-obsidian-implementation-templates.md`
   - Create your first:
     - Daily Dashboard
     - Active Projects Hub
   - Pick 1–2 live projects and build their Project State records

4. **Start Using the Agent Protocol** *(Optional but Recommended)*

   See `12-agent-operating-protocol.md` for how to work with Claude, Grok, or any capable model.

---

5. **Minimum Viable Core**

- One Daily Dashboard
- One Active Projects Hub
- One Project-State note per live project

# Key Concepts

1. **Project State**  
   The single source of truth  
   *(not notes, not folders)*

2. **Canonical Artifact + Supersession**  
   Prevents version drift

3. **Contradictions To Preserve**  
   Prevents premature smoothing

4. **Consequence Binding**  
   Ensures synthesis produces real decisions

5. **Governance Layer**  
   Protects against:
   - False continuity
   - Fluency theater

6. **Restart Cost Reduction**  
   Structures work so projects can survive interruption and resume cleanly

7. **Operational Synthesis**  
   Turns fragmented thinking into durable, actionable execution systems
